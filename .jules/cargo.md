## 2024-06-25 - [Go Module deadcode Unreachable Paths] **Learning:** The standard library or project modules might contain unused utility methods that `deadcode` spots perfectly but which aren't technically errors or problems, just extra exported methods. They are safe to prune to keep the package sizes tight. **Action:** Next time when checking for deadcode in a Go project, carefully check exported methods on common structs or utility wrappers and verify with `grep` if they have any references in tests or actual execution paths.
